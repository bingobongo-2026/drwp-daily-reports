管理ユーザー名 / トークン

このサーバーの管理画面 (/admin/ui/*) と admin API (/admin/*) のアクセスを保護する HTTP Basic Auth のユーザー名とパスワードです。両方とも一致して初めて認証が通ります。

初回は環境変数 DRWP_ADMIN_USERNAME / DRWP_ADMIN_TOKEN で起動時に設定し、運用中はこの画面から差し替える形を推奨します（ユーザー名のデフォルトは admin）。

保存すると Basic 認証の realm が更新されるため、ブラウザはキャッシュした古い認証情報を破棄して再度ダイアログを表示します。

2 段階認証 (TOTP)

Basic 認証だけでは「ユーザー名 + パスワードを 1 セット盗めば終わり」なので、Google Authenticator 系の 6 桁コードを追加で要求します。実機を物理的に持っていない攻撃者は突破できません。

セットアップ時にリカバリーコード 10 個が表示されます (再表示不可)。Authenticator を入れた端末を紛失した時の最後の砦になるので、パスワードマネージャや印刷物としてサーバとは別の場所に保管してください。

緊急時のロックアウト解除: コンテナ環境変数 DRWP_TOTP_DISABLED=1 を立てて再起動すると、2FA ゲートが一時的にオフになります。直ちにログインして 2FA を無効化 → 再セットアップしてください。

署名鍵のローテート

ライセンス検証応答に使う Ed25519 鍵ペアを新しいものに交換します。古い公開鍵は previous_keys として /api/public-key で配布され続けるので、クライアント側は無停止で移行できます。

バックアップ / 復元

zip には signing.key (現在の秘密鍵)、signing.key.previous.json (引退済み公開鍵)、data.sqlite3 (ライセンス DB) の3つが含まれます。月に1回程度、ダウンロードして手元に保管しておけば、コンテナごと消えた場合でも復元できます。

復元時は同名ファイルを上書きします。新たなクライアント検証エラーを避けるため、復元後は管理 UI 上で公開鍵が想定どおりであることを確認してください。
